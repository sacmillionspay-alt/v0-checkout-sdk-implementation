Token de Autenticação

Os tokens de autenticação garantem a identidade do usuário ou aplicação ao acessar as APIs da Millionspay, sem a necessidade de enviar credenciais sensíveis em cada solicitação.

Ambiente Sandbox Testes
  1. 1 Acesse o Portal do Desenvolvedor Millionspay
  2. 2 Clique na aba Tokens
  3. 3 Copie seu token de sandbox
Ambiente de Produção Live
  1. 1 Acesse sua conta Millionspay
  2. 2 No menu lateral, selecione Venda online > Integrações
  3. 3 Clique em Gerar Token

Nota

Algumas APIs requerem, além do token de autenticação, dados como client_id e client_secret, obtidos através da API Connect.

Checkout e Link de Pagamento

A API de Checkout da Millionspay oferece uma solução completa para simplificar o processo de pagamento online. Gere links compartilháveis que direcionam os clientes a uma página de pagamento segura.

Checkout

Redirecione clientes para o ambiente de pagamento seguro.

Link de Pagamento

Compartilhe links em redes sociais, e-mails e mensagens.

Checkout Recorrente

Automatize cobranças para mensalidades e assinaturas.

Ambiente Seguro

Certificação PCI-DSS para máxima segurança.

Meios de Pagamento Disponíveis

Cartão de Crédito Cartão de Débito Pix Boleto Pagar com Millionspay

Como a API de Checkout Funciona

Fluxo de integração passo a passo.

1

Coleta de Informações

Colete as informações dos itens associados ao pedido através do seu e-commerce.

2

Criação do Checkout

Utilize POST /checkouts para criar uma página de pagamento.

3

Recebimento do Link

A API retorna um id e um link de pagamento em links[].href.

4

Redirecionamento

Redirecione o cliente para a página de pagamento utilizando o link fornecido.

5

Pagamento

O cliente conclui o pagamento com o método desejado.

Retorno

Após a conclusão, o cliente é redirecionado para a redirect_url.

Exemplo de resposta com link de pagamento 
{
  "links": [
    {
      "rel": "PAY",
      "href": "https://pagamento.millionspay.com.br/pagamento?code=XXXX",
      "method": "GET"
    }
  ]
}

Personalização da Página de Pagamento

Você pode pular etapas do processo de pagamento fornecendo os dados do cliente antecipadamente.

Parâmetro Tipo Descrição
customer_modifiable Boolean Define se o passo de fornecimento dos dados pessoais será apresentado.
address_modifiable Boolean Define se o passo de fornecimento dos dados do endereço será apresentado.

Cálculo do Valor

valor total = valor dos itens + valor do frete + valor adicional - desconto

Atenção

O valor máximo do carrinho não deve ultrapassar 8999999100 centavos (R$ 89.999.991,00).

Configuração de Parcelamento

Por padrão, as taxas de parcelamento são arcadas pelo comprador. Configure o checkout para assumir essas taxas.

12x com juros grátis nas primeiras 5 parcelas 
{
  "config_options": [
    {
      "option": "INSTALLMENTS_LIMIT",
      "value": "12"
    },
    {
      "option": "INTEREST_FREE_INSTALLMENTS",
      "value": "5"
    }
  ]
}

Checkout Recorrente

Automatize cobranças periódicas como mensalidades, assinaturas ou serviços recorrentes. A primeira cobrança é realizada no momento do pagamento, e uma assinatura é criada automaticamente.

Nota

O Checkout Recorrente está disponível apenas para pagamentos via cartão de crédito.

Parâmetros do Plano

Parâmetro Tipo Descrição
recurrence_plan.name String Nome do plano de cobrança recorrente.
recurrence_plan.interval Object Intervalo de cobrança (unit: MONTH/YEAR, length: número).
recurrence_plan.billing_cycles Integer Número de ciclos de cobrança do plano.
Exemplo de requisição com recorrência 
{
  "reference_id": "ASSINATURA_001",
  "items": [
    {
      "name": "Plano Premium",
      "quantity": 1,
      "unit_amount": 4990
    }
  ],
  "recurrence_plan": {
    "name": "Plano Premium Mensal",
    "interval": {
      "unit": "MONTH",
      "length": 1
    },
    "billing_cycles": 12
  }
}
POST /checkouts

Criar Checkout

Permite criar checkouts personalizados para cada pedido recebido dos seus clientes.

cURL 
curl -X POST https://api.millionspay.com.br/checkouts \
  -H "Authorization: Bearer {seu_token}" \
  -H "Content-Type: application/json" \
  -d '{
    "reference_id": "PEDIDO_001",
    "items": [
      {
        "name": "Produto Exemplo",
        "quantity": 1,
        "unit_amount": 9900
      }
    ],
    "payment_methods": [
      {"type": "CREDIT_CARD"},
      {"type": "PIX"},
      {"type": "BOLETO"}
    ],
    "redirect_url": "https://seusite.com.br/sucesso",
    "notification_urls": ["https://seusite.com.br/webhook"]
  }'

Parâmetros do Body

Parâmetro Tipo Descrição
reference_id String(64) Obrigatório Identificador único atribuído pelo vendedor.
items Array Obrigatório Lista de produtos (name, quantity, unit_amount).
payment_methods Array Opcional Meios de pagamento aceitos.
redirect_url String(255) Opcional URL de redirecionamento após pagamento.
notification_urls Array Opcional URLs para notificações de webhook.
GET /checkouts/{checkout_id}

Consultar Checkout

Permite consultar um checkout e todos os pagamentos a ele associados.

cURL 
curl -X GET https://api.millionspay.com.br/checkouts/CHEC_XXXX \
  -H "Authorization: Bearer {seu_token}"
POST /checkouts/{checkout_id}/inactivate

Inativar Checkout

Desativa um checkout. O checkout continuará existindo, mas o cliente não poderá realizar o pagamento enquanto estiver desativado.

cURL 
curl -X POST https://api.millionspay.com.br/checkouts/CHEC_XXXX/inactivate \
  -H "Authorization: Bearer {seu_token}"
POST /checkouts/{checkout_id}/activate

Ativar Checkout

Ativa um checkout que tenha sido inativado (status INACTIVE).

Atenção

Não é possível ativar um checkout depois que ele expirou.

cURL 
curl -X POST https://api.millionspay.com.br/checkouts/CHEC_XXXX/activate \
  -H "Authorization: Bearer {seu_token}"

Objeto Checkout

Referência completa de todos os parâmetros do objeto Checkout.

Parâmetro Tipo Descrição
id String Identificador único do checkout. Ex: CHEC_XXXX
reference_id String(64) Identificador único atribuído pelo vendedor.
expiration_date ISO-8601 Data de expiração do checkout.
customer Object Dados do comprador (name, email, tax_id, phone).
items Array Lista de produtos (name, quantity, unit_amount).
shipping Object Dados de entrega (type, address, amount).
payment_methods Array Meios de pagamento aceitos.
redirect_url String(255) URL de redirecionamento após pagamento.
notification_urls Array URLs para notificações de webhook.
status ENUM Status: ACTIVE, INACTIVE, EXPIRED.
links Array Links para operações (PAY, SELF, ACTIVATE, INACTIVATE).

Códigos de Erro

Os erros estão normalmente associados ao fornecimento de parâmetros incorretos ou falta de autenticação.

Código Status Descrição
400 Bad Request Requisição inválida - parâmetros incorretos ou ausentes.
401 Unauthorized Token de autenticação inválido ou ausente.
403 Forbidden Sem permissão para acessar o recurso.
404 Not Found Checkout não encontrado.
422 Unprocessable Entity Dados válidos mas não processáveis.
500 Internal Server Error Erro interno do servidor.
